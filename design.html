User Testing Report

For the user testing all team members asked 2 people from our target group to do three different tasks. This means a total of eight people have tested the website before we started building it. The tasks were very diverse and included the most important features of our first prototype of the website.
The three different tasks were the following:

    • Book an activity in the mountains
      Write a review about LeafTime
      Check if there is an activity on the 22nd of April

    • We printed the drawn wireframes on paper and started by giving the user the homepage. Based on the steps the user took we gave him/her the next page. We thought this method would be the most efficient, because it is like testing the real website, but on paper.

    Issues and Improvements

    The main issue we found was that there was no possibility to write a review. Now, reviews can be written and read from the homepage.
    Besides, the main aim of our website is for users to book our activities. Therefore, we added a “BOOK NOW” button to the homepage as well to make the booking process go faster.
    Less clicks to book an activity were recommended by our testers, so we combined the activity page per location in one page.
    The calendar on the side bar is not a necessary feature. It is easier to just put this on the booking page when people are booking their activity.
